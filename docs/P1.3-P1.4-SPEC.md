## Moteur MC — Protection IP

Le moteur de ~4,000 lignes est extrait de `quiz-essentiel.html` lignes 64-2100
et placé dans `/lib/engine/index.ts` côté serveur. Il n'est **jamais** bundled 
côté client. Les API routes l'importent directement.

Le client envoie seulement les réponses quiz (28 champs légers, ~2KB JSON).
Tout le calcul se fait server-side.

## Limites Vercel serverless

- Taille max fonction: 50MB (chromium = ~45MB, ça passe)
- Timeout: 60s (Pro) / 10s (Free) — PDF gen prend ~5-8s
- **IMPORTANT**: Mettre le webhook sur le plan Pro ou configurer 
  `maxDuration: 60` dans la route config

## Flow détaillé

### 1. Quiz → Checkout
Le quiz (client-side) collecte les réponses et POST à `/api/checkout`.
Le serveur crée une session Stripe Checkout avec les quiz answers en metadata.
Le user est redirigé vers Stripe.

### 2. Stripe → Webhook
Après paiement, Stripe envoie `checkout.session.completed` au webhook.
Le webhook extrait les quiz answers, fait tourner le MC, génère le HTML,
le convertit en PDF via Puppeteer, upload sur Blob, et envoie l'email.

### 3. Confirmation
Le user est redirigé vers `/merci` avec son session_id.
La page affiche une confirmation et un lien de téléchargement direct.
